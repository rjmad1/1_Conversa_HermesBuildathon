# Judging Criteria Matrix (submission framework)

> This is a practical scoring framework for judge communication, not an official rubric reproduction.

| Criterion | Conversa evidence | Strength | Limitation | Demo moment |
| --- | --- | --- | --- | --- |
| Problem relevance | Meeting-to-action execution gap | High | Limited real-user telemetry | Intro framing |
| Innovation | Governance-first AI action pipeline | High | Integrations still partial | Scene 2/3 |
| Technical implementation | Hono/Vite, domain/use-case boundaries | Medium-High | In-memory persistence | Architecture section |
| AI usage | Structured extraction + action proposals | High | Model quality variability | Analysis output |
| Security | Isolation remediation + adversarial evidence | High | No production auth | Scene 5 |
| UX | Clear flow for transcript→approval | Medium | Prototype polish | Live walkthrough |
| Business viability | ROI model + use-case fit | Medium | Assumptive economics | ROI section |
| Scalability vision | 3-horizon roadmap | Medium | Future-state only | Roadmap close |
| Partner usage | Transparent status matrix | Medium | Many partner items planned | Partner matrix |
| Demo quality | Stable synthetic path + backup plan | High | Live dependencies may vary | Demo script |
| Completeness | Evidence pack + index + FAQ | High | Public link verification external | Checklist |
| Transparency | Explicit limitations and disclosures | High | Prototype constraints visible | Limitations/Q&A |
