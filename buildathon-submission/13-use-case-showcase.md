# Use-Case Showcase

## 1) Sprint Planning
- **Problem**: decisions and owners lost after planning call.
- **Meeting input**: transcript with commitments and risks.
- **Extracted decision**: sprint scope confirmed.
- **Extracted risk**: dependency uncertainty.
- **Proposed actions**: owner-tagged tasks with due dates.
- **Approval point**: PM approves execution-critical tasks.
- **Business outcome**: cleaner handoff to delivery.
- **Current coverage**: Verified in transcript → analysis → approval path.
- **Future integration**: Jira issue writeback.

## 2) Architecture Decision Meeting
- **Problem**: ADR-level decisions not operationalized.
- **Input**: transcript of trade-off discussion.
- **Decision**: chosen architecture option.
- **Risk**: migration complexity risk.
- **Actions**: design doc update, spike, review.
- **Approval**: engineering lead validates action set.
- **Outcome**: fewer dropped architecture follow-ups.
- **Coverage**: Verified extraction + governance.
- **Future**: repo/PR linkage.

## 3) Incident Postmortem
- **Problem**: corrective actions not consistently tracked.
- **Input**: incident review transcript.
- **Decision**: prevention strategy adopted.
- **Risk**: recurrence window.
- **Actions**: runbook updates, alert tuning.
- **Approval**: SRE lead approval gate.
- **Outcome**: accountable remediation steps.
- **Coverage**: Verified core flow.
- **Future**: ticket + on-call tool integration.

## 4) Customer QBR
- **Problem**: commitments from QBRs are fragmented.
- **Input**: customer meeting transcript.
- **Decision**: roadmap alignment items.
- **Risk**: timeline mismatch.
- **Actions**: owner/date follow-ups.
- **Approval**: account lead approves external commitments.
- **Outcome**: improved follow-up discipline.
- **Coverage**: Verified action extraction/governance.
- **Future**: CRM sync.

## 5) Release-Readiness Review
- **Problem**: release blockers and owners unclear.
- **Input**: release-go/no-go transcript.
- **Decision**: release conditional approval.
- **Risk**: unresolved regression.
- **Actions**: fix owner + validation deadline.
- **Approval**: release manager gate.
- **Outcome**: clearer release accountability.
- **Coverage**: Verified approval/audit pipeline.
- **Future**: CI/CD status integration.
